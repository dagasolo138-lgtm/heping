# 《生灵》项目交接总览（CODEX）

本文件用于防止项目记忆丢失，并帮助新的对话或开发者快速接手。

**维护规则：** 开发前按需查阅；每次有效开发后在末尾追加版本概述；每次影响 Pages 的发布同步更新根目录 `version.json`；修改已有文件前重新获取最新 SHA。

## 当前版本：v0.26.0

《生灵》是部署在 GitHub Pages 的纯前端 ES Module 动态世界模拟游戏。

### 不可破坏规则

- 世界事实优先；人物、传记、史书、关系与未来 AI 只读或解释事实。
- 人物不知玩家或游戏存在。
- 主观印象写入 `personal`；行动、建造、睡眠、播种、收获写入 `lifeEvents`。
- AI 只负责表达，不得决定行动或修改世界状态。
- `globalThis.shengling` 为运行时模块挂接点；异步模块必须展开当前对象，避免覆盖其他模块。
- 读档保持事务性：目标快照先验证；失败时恢复长期状态、建筑瞬时预留和原行动运行时。
- 成功读档采用 `cancel-and-replan`：恢复实时坐标和长期事实，取消未完成任务后重新规划。
- 世界结算必须经过固定 tick；UI 事件不得承担世界事实结算。
- 运行时占用必须进入统一预留账本，并在完成、取消、死亡、路线失败和读档重建时释放。
- `version.json.sourceCommit` 必须指向本次发布的实际功能提交。

## v0.26 模拟内核

### 固定 tick

```text
1 tick = 1 世界分钟 = 1 / 6 模拟秒
```

- `requestAnimationFrame` 只提供现实时间增量。
- `0.5× / 1× / 2× / 5× / 10×` 决定每帧消费的固定 tick 数量。
- 固定结算顺序：

```text
时间推进
→ simulation:pre-tick
→ 季节同步
→ 昼夜 / 天气 / 篝火
→ simulation:tick
→ 生态 / 农田 / 食物损耗 / 道路采样
→ 人物移动与工作
→ 行动规划
→ 人物需求
```

- `simulation:time` 只用于 UI 时间发布，最高每 100ms 一次。
- `actionSystem.advanceTicks(count)` 为无画面确定性回放入口。

### UI 限频

- `createUiRenderScheduler()` 合并同一窗口内的刷新请求。
- 主界面、农业、扩田、生态、道路和食物读数最高 10 FPS。
- 高频世界事件不得直接重复执行完整 DOM 重绘。
- `window.shengling.uiScheduler.getDiagnostics()` 可查看请求数、渲染数和待处理原因。

### 统一预留账本

`src/modules/actions/reservationLedger.js` 当前记录：

- `task-slot`：行动并发名额。
- `feature`：树木、浆果等目标物件。
- `camp-storage`：搬运任务占用的营地剩余容量。
- `building-material`：工地材料预留的运行时映射。

运行时 API：

```js
window.shengling.reservationLedger.list()
window.shengling.reservationLedger.getSummary()
window.shengling.actionSystem.getDiagnostics().reservations
```

账本是瞬时运行时状态，不写入长期世界存档。成功读档重建代理时清空；失败读档保留原行动运行时与账本。

### 可复现 ID 与人物运行时视图

- `createId()` 使用世界命名空间种子和递增序列生成 ID。
- `resetIdSequence(seed)` 用于测试和新世界初始化。
- 建造、农事、天气、睡眠任务不再使用 `crypto.randomUUID()`。
- `peopleSystem.getRuntime()` 与 `getAliveRuntime()` 排除不断增长的记忆数组，供固定 tick 高频读取。
- `actionSystem.getMovementPeople()` 只暴露道路采样所需的 ID、坐标和行动状态。
- 长期人物查询和 UI 仍使用完整人物对象。

### 第 30 日确定性基线

固定种子：

```text
replay-seed-v026
```

目标时刻：

```text
生灵历 1 年第 30 日 12:00
42,000 fixed ticks
```

SHA-256 世界指纹：

```text
0876a28dbd85d9ceb38c273a2049bb9433439089169be8bb27f79037efb73c4c
```

指纹覆盖人物状态与坐标、库存、关系计数、建筑、篝火、农田、生态、道路、食物储存、社会事件、史书、地图物件、运行时预留和近期日志。

### CI

`.github/workflows/ci.yml` 在 PR 和 `main` 推送时执行：

```text
npm ci
npm run check
npm test
第 30 日指纹提取
npm run build
```

CI 会把指纹写入 `day30-<fingerprint>` 工件名称，便于核验基线。

## 已知限制

1. 存档为浏览器本地 `localStorage`，没有跨设备云存档。
2. 成功读档不续接路径游标、工作耗时或中途动画。
3. 当前预留账本尚未形成完整资源流水与每日收支报表。
4. 食物分配规则除 `firstComeFirstServed` 外仍以解释和记录为主。
5. 草棚、储物棚和农田尚未成为寻路障碍。
6. 农业、天气、食物损耗和劳动参数尚未完成长期平衡。
7. 真实 iPhone 安全区、浏览器地址栏变化和极小屏仍需人工回归。

# 《生灵》项目交接总览（CODEX）

本文件用于防止项目记忆丢失，并帮助新的对话或开发者快速接手。

**维护规则：** 开发前按需查阅；每次有效开发后在末尾追加版本概述；每次影响 Pages 的发布同步更新根目录 `version.json`；修改已有文件前重新获取最新 SHA。

---

## 当前版本：v0.25.3

《生灵》是部署在 GitHub Pages 的纯前端 ES Module 动态世界模拟游戏。

### 不可破坏规则

- 世界事实优先；人物、传记、史书、关系与未来 AI 只读或解释事实。
- 人物不知玩家或游戏存在。
- 主观印象写入 `personal`；行动、建造、睡眠、播种、收获写入 `lifeEvents`。
- `globalThis.shengling` 为运行时模块挂接点；异步模块必须展开当前对象，避免覆盖其他挂接模块。
- 已有最小持久化；当前开发提交无法预知自身最终 SHA，`version.json.sourceCommit` 使用明确发布占位说明，发布后必须回填实际 commit SHA，不得保留占位值。
- 读档必须保持事务性：目标快照先完成结构与导入目标验证；任何子系统导入失败都要尝试恢复读取前快照、工地瞬时预留与原行动运行时，并按读取前状态恢复行动循环。
- 任务中途存档采用 `cancel-and-replan`：保存实时小数坐标和被中断任务摘要；成功读档恢复位置后取消半完成任务，不延续路径游标或工作计时。

### 当前能力

- 十位村民、河谷地图、A* 寻路、采集、营地库存、草棚、储物棚、昼夜、天气、篝火、环境暴露。
- 最小持久化：世界时间、人物、行动运行时坐标、地图、营地、规则、建筑、篝火、生态、道路、农田、食物储存、食物分配冷却、社会事件与史书可保存/读取。
- 事务化读档：导入前验证目标系统，导入失败后恢复读取前快照、建筑瞬时检查点和原行动运行时，发出 `save:load-failed` 诊断事件，并恢复读取前的模拟运行状态。
- 运行时恢复：存档记录人物移动中的实时小数坐标与被中断任务摘要；读档后回到保存位置，活动转为空闲并重新规划。
- 资源守恒：已领到人物背包的建材随人物存档保留；普通建筑存档剥离瞬时预留，失败回滚检查点保留 `reserved/carried` 预留。
- 效用评分：通用行动按生存、库存、季节、关系与候选收益评分，人物面板展示行动原因。
- 关系反馈：帮助意图与共同劳动会产生关系变化和每日去重记忆，覆盖普通劳动、建造、开田、播种、收获与运料。
- 冬季稀缺：季节影响浆果、木材、燃料消耗和储备目标。
- 营地规则：记录食物分配、篝火和储物访问规则；当前食物规则为 advisory/explanatory，除先到先得外尚不改变即时发放顺序。
- 事件传播：资源分配/拒绝、稀缺、关系变化、规则变化、建筑完成和重大收获会形成亲历/听闻；普通劳动只保留 lifeEvents。
- 规则史书：建筑、农田、资源拒绝、规则变化和重大社会事件写入聚落纪事。
- LLM 只读边界：仅导出 fact pack 供外部润色/问答，不接入行动决策，不允许写回世界状态。
- 资源恢复、踩踏路径/土路、食物批次与腐败、四季、粟米、两块农田、土壤肥力。
- 地图支持拖动、双指缩放、滚轮、键盘、Home 回营地与人物点选。
- 桌面端采用地图主视图与右侧观察抽屉；人物、营地、事件、纪事使用同一组页签，避免信息与地图争抢空间。
- 手机端观察抽屉默认收起为底部操作条；把手支持点击、上滑展开和下滑收起，选人自动打开人物页；系统菜单统一收纳存档与构建信息。
- 世界速度：0.5×、1×、2×、5×、10×，默认 1×。
- 营地满载时，携带物资的村民等待空位，避免无效搬运循环与继续叠加采集。
- 高倍速时路径采样随速度缩短，减少转弯路径被直线补全。
- 建造事实按实际建筑与材料写入，不再把储物棚记成草棚。

### 已知限制

- 当前世界存档仍是浏览器本地 `localStorage`，不是云存档；跨设备需要手动迁移浏览器数据。
- 读档恢复精确位置，但不会延续原任务的路径游标、工作计时或半完成阶段；当前策略是取消后重新规划。
- 食物分配规则除 `firstComeFirstServed` 外目前只参与解释、优先级展示和事件记录，尚未实现同 tick 排序队列。
- 移动端抽屉已有纯函数和指针事件回归测试；真实 iPhone 的安全区、浏览器地址栏变化和极小屏可读性仍需人工回归。

## 世界速度模型

允许倍率：`0.5× / 1× / 2× / 5× / 10×`。

基础速率 `WORLD_MINUTES_PER_REAL_SECOND = 6`：

| 倍率 | 世界分钟 / 现实秒 |
|---:|---:|
| 0.5× | 3 |
| 1× | 6 |
| 2× | 12 |
| 5× | 30 |
| 10× | 60 |

```text
realDelta = clamp((now - previous) / 1000, 0, 0.12)
simulationDelta = realDelta × 当前倍率
```

`simulationDelta` 推进世界时间、人物移动/工作、行动规划、人物需求、环境暴露、天气、篝火、季节、生态、作物、土壤和食物损耗。高倍速不只是时钟加速。

`simulation:speed` 事件：

```text
{ speed: { value, label, worldMinutesPerRealSecond }, previous, reason, time }
```

运行时 API：

```js
window.shengling.worldSpeedSystem.get()
window.shengling.worldSpeedSystem.set(5)
window.shengling.worldSpeedRuntime.set(2)
```

速度不持久化；刷新恢复 1×；暂无暂停 0×。

### 高倍速路径规则

路径采样基准为 240ms 现实时间：

```text
sampleInterval = max(16ms, 240ms / 当前倍率)
```

这使 10× 下单次采样的移动距离与 1× 接近，降低折线路径被直线补全的概率。

## 库存与建造防护

- `availableCampStorage(camp, storage)` 优先读取 `storage.available`；没有快照时用 `capacity - items 总量` 推导。
- 人物携带物资时，若可用容量为 0，`planNextAction()` 返回空任务，人物等待容量释放。
- 营地因进食、施工领料或食物腐败释放容量后，后续规划自动恢复搬运。
- 建造领料、运输、施工文本必须从 `buildingSystem.get(siteId).label` 获取实际建筑名称。
- 普通建筑存档不得保留瞬时材料预留；事务回滚检查点必须保留 `reserved/carried` 预留，失败读档不得使原任务失去资源占用。
- 成功读档取消运料任务时，已从营地领出的材料保留在人物背包，后续由通用搬运规划归还营地，不直接写入工地或凭空删除。

## 农业关键参数

- 一年 360 天；春夏秋冬各 90 天。
- 粟米：春可播，夏 ×1.25，秋 ×0.62，冬 ×0；成熟全年可收。
- 基础收获 8，返种 2，收获肥力 -18。
- 土壤：初始 78；休耕每世界分钟 +0.0015；成长倍率 1.06/0.96/0.78/0.60，产量倍率 1.12/1.00/0.80/0.62。
- 第二块田由第一块田首次收获解锁，开垦需要 10 工作量。

## 移动端规则

- 观察抽屉默认收起，底部必须保留把手与页签；点击把手、点击页签、地图选人或向上滑动把手会展开，向下滑动把手会收起。
- 手势只在抽屉把手区域接管触控；纵向位移至少 42px，且纵向距离必须达到横向距离的 1.2 倍。
- 指针移动超过 8px 且以纵向为主时阻止默认滚动，并吞掉手势结束后的合成点击，避免抽屉完成滑动后立刻反向切换。
- 把手使用 `touch-action: pan-x`，地图 canvas 继续保留原有单指页面滚动、双指缩放和显式拖动模式，不共享手势状态。
- `map-canvas-wrap.is-hud-collapsed` 时只保留核心地图状态，点击“地图信息”再展开季节、生态、路径、农事、扩田与食物信息。
- 人物、营地、事件、纪事共用底部抽屉，不在地图下方重复堆叠长列表。
- 倍速选择器保持可达；系统菜单承载存档与构建信息，不能遮挡地图交互。
- 触控目标不小于 40px，并尊重底部安全区和 `prefers-reduced-motion`。


## 已知限制

1. 存档为浏览器本地 `localStorage`，不是云存档；发布后仍需用实际 commit SHA 回填 `version.json.sourceCommit` 并验证 Pages 读数。
2. 读档恢复人物精确小数坐标，但不会恢复正在执行任务的路径游标、工作计时和半完成阶段；采用 `cancel-and-replan`。
3. 食物分配规则除 `firstComeFirstServed` 外当前为 advisory/explanatory，尚未真正改变即时请求的发放顺序。
4. 草棚、储物棚、农田仍未作为寻路障碍。
5. 农业只有两块田和一种粟米。
6. 扩田使用固定候选点，没有玩家点选、地形评分或劳动力预算。
7. 肥力、天气、季节、食物损耗和高倍速参数尚未平衡测试。
8. 10× 会更快产生日志和状态刷新；暂无日志聚合或后台跳帧优化。
9. 移动端抽屉已有自动化手势回归；真实 iPhone 的安全区、浏览器地址栏变化和极小屏可读性仍待人工验证。

- `planNextAction()` 在保留携带物资、极端饥渴和旧 planner 回退的前提下，先为取水、采集、砍树、休息生成候选并按效用分选择。
- 被效用 planner 选中的任务会携带 `data.utility` 调试摘要，为下一轮人物面板评分展示做准备。

### v0.17.1 · 行动评分调试面板
- 人物当前行动视图保留 `utility` 调试信息，人物详情面板新增“行动原因”卡片。
- 效用 planner 选中的取水、采集、砍树、休息会显示总分、主要因素和候选评分；搬运和紧急生存行为也会给出结构化原因。
- 取暖任务写入受寒/潮湿评分摘要，为天气压力下的行动解释做准备。

### v0.18.0 · 关系反哺行动评分
- 社会效用开始读取亲属、伴侣、信任和好感关系：亲近者饥渴较高时，取水和采集会得到额外评分。
- 关系紧张者位于候选行动目标附近时，行动评分会受到避让惩罚。
- 通用劳动完成时，若亲近者正在附近执行同类工作，会小幅提升双方熟悉、好感和信任，为共同劳动关系闭环打底。

### v0.18.1 · 关系事件与共同记忆
- 社会效用评分会把被帮助对象写入 `utility.socialTargets`，行动完成后形成帮助意图记忆和 `social:relation-changed` 事件。
- 共同劳动关系反馈现在会写入双方 `personal` 记忆，并按每日/行动去重，避免同一天同类劳动反复刷记忆。
- 关系反馈事件包含 actor、target、action、delta、source 和 time，为后续事件传播系统提供结构化输入。

### v0.19.0 · 冬季稀缺参数
- 生态恢复加入季节倍率：浆果和树木在冬季恢复显著变慢，恢复队列记录 depletion 时的季节和恢复时长。
- 冬季采集浆果产量降为 25%，秋季降为 65%，行动任务会记录季节产量倍率。
- 冬季食物与木材目标提高，效用 planner 会在冬季更重视粮食和燃料储备；冬季寒冷/冷雨下篝火燃料消耗加快。

### v0.19.1 · 食物分配与稀缺事件
- 新增食物/饮水分配系统，当前规则为 `firstComeFirstServed`，保留后续 campRules 接管入口。
- 饥渴结算改为通过分配系统请求水和食物，成功时发出 `survival:resource-distributed`，失败时按冷却发出 `survival:resource-denied` 与 `survival:scarcity`。
- 食物不足不再静默失败；分配失败会保留饥饿并增加少量压力，让冬季储备不足更容易自然形成危机。

### v0.20.0 · 营地规则底座
- 新增 `campRulesSystem`，把食物分配、篝火和储物访问规则记录为可导出、可导入的数据。
- 食物/饮水分配会读取当前食物分配规则，并在分配/拒绝事件中写入规则、优先分和解释，为后续平均分配、劳动者优先、健康低者优先、亲属优先和地位优先提供入口。
- 世界存档纳入营地规则快照，读取存档时会恢复规则修订、历史和当前规则，避免刷新后营地习俗丢失。

### v0.21.0 · 事件传播与共同记忆
- 新增纯规则 `socialEventSystem`：行动完成、分配成功/拒绝、关系变化和规则变化会形成结构化社会事件。
- 同地点附近人物会获得“亲眼所见”个人记忆；见证者会按关系、社交技能和事件严重度把事件传播为“听闻”记忆，且使用稳定哈希概率保证可复现。
- 社会事件和传播记录纳入世界存档；人物详情新增“亲历与听闻”列表，区分亲历事件和传闻。

### v0.22.0 · 规则史书系统
- 新增纯规则 `chronicleSystem`：建筑完成、农田成熟/收获、食物分配拒绝、营地规则变化和重大社会事件会被收集为结构化史实。
- 每十个世界日自动生成一条锁定的聚落纪事；建筑完成、资源拒绝和规则变化会生成即时大事记，史书条目只基于已记录事实。
- 聚落纪事纳入世界存档并在页面中展示，刷新读取后保留既有史书，不使用 LLM 生成或改写世界事实。

### v0.23.0 · LLM 只读解释层边界
- 新增 `llmBoundary`、`factPackBuilder` 与 `aiPolicy`，只导出人物传记、锁定史书润色和人物已知记忆问答所需的 fact pack。
- fact pack 明确禁止编造事实、改写世界状态或决定行动；提示词导出只面向外部 LLM 润色/解释，不接入决策层。
- 输出校验要求 `usedFactIds` 必须来自 fact pack，且拒绝 `statePatch`、`worldPatch`、`nextAction` 等越界写入字段。

- 世界快照导入前先验证顶层结构和所有非空子系统是否具备 `importState`，预验证失败时不会停止模拟或修改世界。

### v0.25.2 · 移动端抽屉手势

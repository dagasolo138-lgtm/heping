# 生灵

《生灵》是一个运行于 GitHub Pages 的动态世界模拟游戏。世界先记录客观事实，再由传记、史书和对话系统解释这些事实。

## 开发交接规则

开发前可按需查阅 [`codex.md`](./codex.md)。**每次完成有效更新后，必须在 `codex.md` 末尾追加版本更新概述；每次影响 Pages 的发布还必须同步更新根目录 `version.json`。**

## 当前阶段：确定性模拟内核 v0.26.0

### 当前能力

- 10 位村民，拥有年龄、职业、技能、性格、家庭关系、库存、事实与个人记忆。
- 起始河谷：160 米 × 120 米；1 格 = 1 米 × 1 米；包含河流、林地、石滩、营地和资源物件。
- 村民会取水、采集浆果、砍树、搬运资源、休息、建造、添柴、取暖、睡眠、开垦、播种与收获。
- A* 网格寻路；资源流为“地图 → 人物背包 → 营地库存 → 工地、篝火或消耗”。
- 草棚建成后自动规划简易储物棚；储物棚增加容量并提供食物储存保护。
- 世界包含昼夜、天气、篝火、潮湿、受寒、四季、生态恢复、踩踏路径、粟田、扩田和土壤肥力。
- 世界速度支持 **0.5×、1×、2×、5×、10×**。
- 地图支持鼠标与触控操作；桌面端使用地图主视图和右侧观察栏，手机端使用底部观察抽屉。
- 存档支持事务化导入、失败回滚、精确运行时坐标恢复和 `cancel-and-replan` 任务中断策略。

### v0.26 确定性内核

- 模拟固定为 **每 tick 推进 1 个世界分钟**，对应 `1/6` 模拟秒。现实帧率只影响每帧消费多少 tick，不改变世界结算顺序。
- 新增 `simulation:pre-tick` 与 `simulation:tick`。季节、天气、篝火、生态、农田、食物损耗、道路采样、行动、需求和规划按固定顺序结算。
- `simulation:time` 只承担 UI 时间发布，最高每 100ms 一次。
- 主界面、农业、扩田、生态、道路和食物读数使用合并调度，最高 **10 FPS**，高倍速不再按世界分钟重复刷新 DOM。
- 新增统一运行时预留账本，覆盖任务名额、资源物件、营地容量和工地材料映射。
- 任务完成、取消、人物死亡、路线失败和读档重建会释放对应预留。
- 人物、任务、建筑、记忆和日志 ID 使用可重置的世界种子序列生成。
- 高频行动结算读取轻量人物运行时视图，避免反复深拷贝不断增长的记忆数组。
- 固定种子完整回放到第 30 日，并校验 SHA-256 世界指纹：

```text
0876a28dbd85d9ceb38c273a2049bb9433439089169be8bb27f79037efb73c4c
```

- GitHub Actions 自动执行语法检查、全量 Node 测试、指纹校验和 Vite 生产构建。

## 目录

```text
version.json      # GitHub Pages 部署清单与构建标记
.github/workflows # CI：语法、测试、确定性回放与生产构建
src/
  core/
    simulation/   # 固定步长时钟
    ui/           # UI 合并与限频调度
    ids/          # 可复现 ID 序列
  modules/
    people/       # 人物、轻量运行时视图、关系、库存与记忆
    map/          # 地图生成、查询与服务层
    actions/      # 规划、执行、固定 tick 与统一预留账本
    settlements/  # 营地库存、容量、规则与储存升级
    buildings/    # 工地、施工、居住与储物棚
    environment/  # 昼夜、天气、篝火与暴露
    ecology/      # 树木与浆果恢复
    roads/        # tick 驱动的踩踏与土路
    seasons/      # 四季日历与农业窗口
    farming/      # 粟米、农田、扩田与土壤
    storage/      # 食物批次、新鲜度与损耗
    persistence/  # 世界快照、运行时坐标与事务化读档
  bootstrap/      # 浏览器运行时扩展和各模块 UI
  ui/map/         # 地图渲染与交互
  styles/         # 桌面与移动端样式
```

## 已知限制

- 存档保存在当前浏览器，没有跨设备云存档。
- 读档恢复人物精确位置，但不会续接原任务的路径游标、工作耗时或中途动画；未完成任务会取消后重新规划。
- 当前账本统一了运行时占用，但尚未形成完整资源收支流水和每日结算报表。
- 草棚、储物棚和农田尚未成为寻路障碍。
- 农业、天气、损耗和劳动参数仍需要长期平衡。
- 真实 iPhone 的安全区、浏览器地址栏变化和极小屏可读性仍需人工回归。

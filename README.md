# 生灵

《生灵》是一个运行于 GitHub Pages 的动态世界模拟游戏。世界先记录客观事实，再由传记、史书和对话系统解释这些事实。

## 开发交接规则

开发前可按需查阅 [`codex.md`](./codex.md)。**每次完成有效更新后，必须在 `codex.md` 末尾追加版本更新概述；每次影响 Pages 的发布还必须同步更新根目录 `version.json`。**

## 当前阶段：生存经济 · 工具与耐久 v0.27.3

### 当前能力

- 10 位村民，拥有年龄、职业、技能、性格、家庭关系、库存、事实与个人记忆。
- 起始河谷为 160 米 × 120 米，包含河流、林地、石滩、营地和资源物件。
- 村民会取水、采集浆果、砍树、搬运资源、休息、建造、添柴、取暖、睡眠、开垦、播种与收获。
- A* 网格寻路；资源流为“地图 → 人物背包 → 营地库存 → 工地、篝火或消耗”。
- 草棚建成后自动规划简易储物棚；储物棚增加容量并提供食物储存保护。
- 世界包含昼夜、天气、篝火、潮湿、受寒、四季、生态恢复、踩踏路径、粟田、扩田和土壤肥力。
- 世界速度支持 **0.5×、1×、2×、5×、10×**。
- 地图支持鼠标与触控操作；桌面端使用地图主视图和右侧观察栏，手机端使用底部观察抽屉。
- 存档支持事务化导入、失败回滚、精确运行时坐标恢复和 `cancel-and-replan` 任务中断策略。

## v0.26 确定性内核

- 模拟固定为 **每 tick 推进 1 个世界分钟**，对应 `1/6` 模拟秒。现实帧率只影响每帧消费多少 tick，不改变世界结算顺序。
- 季节、天气、篝火、生态、农田、食物损耗、道路、行动、需求和规划按固定顺序结算。
- `simulation:time` 只承担 UI 时间发布，主要界面最高 **10 FPS**。
- 统一运行时预留账本覆盖任务名额、资源物件、营地容量和工地材料映射。
- 人物、任务、建筑、记忆和日志 ID 使用可重置的世界种子序列生成。
- GitHub Actions 自动执行语法检查、全量 Node 测试、第 30 日指纹校验和 Vite 生产构建。

## v0.27.1 动态目标库存

营地按人口、季节、天气、腐败、储物保护、燃料和未完成建造计算未来三日安全库存。

```text
有效库存 = 营地现货 + 人物背包 + 在途采集 - 已承诺物资
```

库存目标满足后，非紧急村民不会继续重复取水、采集或伐木；严重饥渴仍保留个人生存例外。

运行时检查：

```js
window.shengling.stockTargetSystem.get()
window.shengling.stockTargetSystem.refresh()
```

## v0.27.2 劳动成本模型

采集、搬运、施工、农事和添柴使用统一劳动成本：

```text
预计劳动成本 = 路程 + 工作耗时 + 额外精力消耗
```

模型读取：

- **距离**：候选阶段进行直线预估，任务分配后使用实际 A* 路线生成精确快照。
- **负重**：水、木材、浆果和粟米按不同重量降低移动速度，并提高搬运能耗。
- **地形**：高草、林地、沙岸、石滩和农田降低速度；泥土地形略快。
- **道路**：磨损小径提供 `1.07×` 移动收益，土路提供 `1.16×`。
- **天气**：降雨、寒冷和冷雨分别降低移动与工作效率。
- **体力**：低精力继续降低移动速度，并延长工作耗时、提高额外能耗。
- **技能**：捕鱼、采集、伐木和建造技能降低对应劳动的额外能耗。
- **行动强度**：砍树、施工和开垦比取水、播种等行动消耗更高。

任务分配时会写入可解释快照：

```js
task.data.laborCost
```

其中包含距离、负重、地形、道路、天气、体力、技能、预计耗时和预计精力。效用评分新增 `laborCost` 因子，村民会在同类可行任务中倾向选择净成本更低的目标。

浏览器运行时诊断：

```js
window.shengling.laborCostSystem.estimate(personId, task)
window.shengling.laborCostSystem.getRecent(10)
```

额外精力在任务阶段内累计、完成时一次结算，避免 10× 下高频人物状态事务。移动热路径只读取原始地形编号、道路倍率和任务内负重快照。

## v0.27.3 工具与耐久

营地拥有四件公共工具：

| 工具 | 当前用途 | 劳动效果 |
|---|---|---|
| 石斧 | 伐木 | 缩短砍伐时间并降低额外能耗 |
| 搬运篮 | 搬运资源、运送建材 | 降低有效负重、移动与搬运能耗 |
| 简易农具 | 开垦、播种、收获 | 缩短农事耗时并降低能耗 |
| 石镐 | 后续采石接口 | 当前进入工具库但尚无可执行采石任务 |

工具属于营地公共资产。任务分配时工具进入统一预留账本：

```text
type = tool
key = toolId
capacity = 1
```

同一件工具不能被两个人同时使用。任务完成后扣除对应磨损；取消、死亡、路线失败或孤立任务只释放占用。耐久归零后工具进入 `broken` 状态并退出任务候选，修理或替换后恢复使用。

工具耐久和损坏状态进入世界存档，瞬时任务占用不写入长期存档。旧版存档没有工具字段时会恢复初始公共工具；失败读档会通过检查点恢复原耐久与运行时占用。

浏览器运行时：

```js
window.shengling.toolSystem.list()
window.shengling.toolSystem.getSummary()
window.shengling.toolSystem.getAssignments()
window.shengling.toolSystem.repair(toolId, amount)
window.shengling.toolSystem.replace(toolId)
```

营地面板会显示每件工具的当前耐久以及“可用 / 使用中 / 损坏”状态。

第 30 日确定性世界指纹继续为：

```text
20b2e6bea8c6f87cde6ee663ffe19ed97dedeb670679a5a7007ca6e4e412461c
```

该基线验证无工具运行时的确定性内核保持不变；工具生命周期由专项测试覆盖。

## 目录

```text
version.json      # GitHub Pages 部署清单与构建标记
.github/workflows # CI：语法、测试、确定性回放与生产构建
src/
  core/
    simulation/   # 固定步长时钟
    ui/           # UI 合并与限频调度
    ids/          # 可复现 ID 序列
  modules/
    people/       # 人物、轻量运行时视图、关系、库存与记忆
    map/          # 地图生成、查询与原始地形服务
    actions/      # 规划、动态库存、劳动成本、固定 tick 与预留账本
    tools/        # 公共工具、耐久、损坏、修理和工具占用
    settlements/  # 营地库存、容量、规则与储存升级
    buildings/    # 工地、施工、居住与储物棚
    environment/  # 昼夜、天气、篝火与暴露
    ecology/      # 树木与浆果恢复
    roads/        # tick 驱动的踩踏、小径与土路
    seasons/      # 四季日历与农业窗口
    farming/      # 粟米、农田、扩田与土壤
    storage/      # 食物批次、新鲜度与损耗
    persistence/  # 世界快照、运行时坐标与事务化读档
  bootstrap/      # 浏览器运行时扩展和各模块 UI
  ui/map/         # 地图渲染与交互
  styles/         # 桌面与移动端样式
```

## 已知限制

- 存档保存在当前浏览器，没有跨设备云存档。
- 读档恢复人物精确位置，但不会续接原任务的路径游标、工作耗时或中途动画；未完成任务会取消后重新规划。
- 动态库存参数、劳动成本系数和工具耐久仍是原型平衡值，需要第 60 日长期回归校准。
- 候选评分使用直线成本预估；任务真正分配后才使用 A* 路线生成精确劳动快照。
- 修理与替换 API 已存在，但尚未形成需要材料、劳动时间和资源流水的正式维修任务。
- 石镐已进入工具目录，采石行动和石材资源尚未实现。
- 当前账本尚未形成完整资源收支流水和每日结算报表。
- 草棚、储物棚和农田尚未成为寻路障碍。
- 真实 iPhone 的安全区、浏览器地址栏变化和极小屏可读性仍需人工回归。

# 生灵

《生灵》是一个运行于 GitHub Pages 的动态世界模拟游戏。世界先记录客观事实，再由传记、史书和对话系统解释这些事实。

## 开发交接规则

开发前可按需查阅 [`codex.md`](./codex.md)。**每次完成有效更新后，必须在 `codex.md` 末尾追加版本更新概述；每次影响 Pages 的发布还必须同步更新根目录 `version.json`。**

## 当前阶段：人物 + 地图 + 行动 + 建造 + 居住 + 环境 + 储物 + 生态 + 路径 + 农田 + 损耗 + 季节 + 人工扩田 + 土壤肥力 + 可验证部署 v0.14

### 当前能力

- 10 位村民，拥有年龄、职业、技能、性格、家庭关系、库存、事实与个人记忆。
- 起始河谷：160 米 × 120 米；1 格 = 1 米 × 1 米；河流、林地、石滩、营地和资源物件。
- 村民会取水、采集浆果、砍树、搬运资源、休息、建造、添柴、取暖、睡眠、开垦、播种与收获。
- A* 网格寻路；资源流为“地图 → 人物背包 → 营地库存 → 工地、篝火或消耗”。
- 草棚建成后自动规划简易储物棚；营地初始 24 单位容量，储物棚建成后增加 72 单位容量并提供储存保护。
- 世界包含昼夜、晴朗、阴天、降雨、寒冷、冷雨；天气影响效率，篝火消耗木材，草棚和火源减少环境暴露。
- 树被砍伐后留下树桩，3 个世界日后恢复；浆果被采空后留下恢复灌丛，1 个世界日后恢复。建筑占用位置会让恢复延后。
- 村民反复经过同一地块会积累踩踏痕迹；4 次经过出现路径，10 次经过形成土路。成型土路提供 16% 移动速度加成。
- 储物棚建成后自动选择第一块粟田。村民开垦、播种粟米；成熟后收获粟米并留下下一轮种子。
- 一年固定 360 天，春、夏、秋、冬各 90 天。季节会给天气叠加温度趋势，并影响粟米的农业窗口：春季可播种，夏季成长加速，秋季成长减慢，冬季停长；成熟作物全年可收获。
- 第一块粟田首次收获后，会自动在营地周边选出第二块人工扩田。第二块田需要 10 点开垦工作量，之后复用现有的春播、季节成长、收获与粮食储存链路。
- 每块田拥有独立土壤肥力：新田初始 78；休耕时缓慢恢复；肥力会影响作物成长和收获量；每次粟米收获消耗 18 点肥力。地图以土色和状态点显示肥力等级，农事读数显示平均肥力。
- 地图支持鼠标/单指拖动、双指缩放、滚轮缩放、按钮缩放、方向键平移、`+/-` 缩放和 Home 回营地；点击村民可直接查看其状态。移动端村民列表改为横向卡片滚动。
- 页面右下角显示构建版本、构建编号与源码短提交号，并提供“部署清单”入口。页面读取清单时使用缓存穿透，便于确认 GitHub Pages 是否已切到最新版本。
- 浆果和粟米会按进入营地的批次保存。每批有独立新鲜度；村民优先吃最不新鲜的批次。降雨和冷雨加快损耗，储物棚保护值降低损耗；完全腐败的食物会从营地库存移除并计入累计损耗。

## 目录

```text
version.json      # GitHub Pages 部署清单与构建标记
src/
  core/          # ID、事件总线、游戏时间、确定性随机数
  data/          # 常量和基础数据
  modules/
    people/       # 人物系统
    map/          # 地图数据、生成、查询与服务层
    settlements/  # 营地库存、容量、食物批次与储存升级
    actions/      # 任务规划、寻路、执行、睡眠、添柴与农事调度
    buildings/    # 建筑目录、工地、施工、居住与储物棚
    environment/  # 昼夜、天气、篝火、环境暴露
    ecology/      # 树木与浆果恢复队列
    roads/        # 踩踏热度与土路速度效果
    seasons/      # 四季日历、温度趋势与农业窗口
    farming/      # 粟米、农田、扩田、土壤模型、作物生长与收获
    storage/      # 食物批次、新鲜度、天气损耗与储物保护
  bootstrap/      # 运行时扩展模块挂接，含部署构建信息读取
  ui/map/         # 地形、农田、路径、光照、天气、物件、建筑、人物令牌与交互
  styles/         # 页面样式，含移动端地图交互体验层
```

## 运行

这是纯前端 ES Module 项目。GitHub Pages 发布后直接访问 `index.html` 即可。

## 部署确认

每次有效发布都应更新根目录 `version.json` 的 `version`、`buildId` 与 `sourceCommit`。

1. 打开 Pages 页面，确认右下角构建读数显示预期版本和构建编号。
2. 点击右下角“部署清单”，或访问 `https://dagasolo138-lgtm.github.io/heping/version.json?cacheBust=任意新数字`。
3. 对比 `buildId` 和 `sourceCommit`。两处一致，且与本次发布清单一致，即可确认线上页面读取到了本次构建。

`version.json` 的缓存参数与页面内 `cache: 'no-store'` 请求共同避免浏览器缓存把旧部署误判为新部署。

## 已知限制

刷新页面会重开局；没有浏览器端自动化测试；草棚、储物棚、农田未作为寻路障碍；路径、生态、农田、食物批次不持久化。土壤恢复、作物成长、收获、扩田阈值与天气倍率仍是原型参数，尚未完成平衡测试。

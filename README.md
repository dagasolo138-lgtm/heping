# 生灵

《生灵》是一个运行于 GitHub Pages 的动态世界模拟游戏。世界先记录客观事实，再由传记、史书和对话系统解释这些事实。

## 开发交接规则

开发前可按需查阅 [`codex.md`](./codex.md)。**每次完成有效更新后，必须在 `codex.md` 末尾追加版本更新概述；每次影响 Pages 的发布还必须同步更新根目录 `version.json`。**

## 当前阶段：生存经济 · 动态目标库存 v0.27.1

### 当前能力

- 10 位村民，拥有年龄、职业、技能、性格、家庭关系、库存、事实与个人记忆。
- 起始河谷为 160 米 × 120 米，包含河流、林地、石滩、营地和资源物件。
- 村民会取水、采集浆果、砍树、搬运资源、休息、建造、添柴、取暖、睡眠、开垦、播种与收获。
- A* 网格寻路；资源流为“地图 → 人物背包 → 营地库存 → 工地、篝火或消耗”。
- 草棚建成后自动规划简易储物棚；储物棚增加容量并提供食物储存保护。
- 世界包含昼夜、天气、篝火、潮湿、受寒、四季、生态恢复、踩踏路径、粟田、扩田和土壤肥力。
- 世界速度支持 **0.5×、1×、2×、5×、10×**。
- 地图支持鼠标与触控操作；桌面端使用地图主视图和右侧观察栏，手机端使用底部观察抽屉。
- 存档支持事务化导入、失败回滚、精确运行时坐标恢复和 `cancel-and-replan` 任务中断策略。

## v0.26 确定性内核

- 模拟固定为 **每 tick 推进 1 个世界分钟**，对应 `1/6` 模拟秒。现实帧率只影响每帧消费多少 tick，不改变世界结算顺序。
- 季节、天气、篝火、生态、农田、食物损耗、道路、行动、需求和规划按固定顺序结算。
- `simulation:time` 只承担 UI 时间发布，主要界面最高 **10 FPS**。
- 统一运行时预留账本覆盖任务名额、资源物件、营地容量和工地材料映射。
- 人物、任务、建筑、记忆和日志 ID 使用可重置的世界种子序列生成。
- GitHub Actions 自动执行语法检查、全量 Node 测试、第 30 日指纹校验和 Vite 生产构建。

## v0.27.1 动态目标库存

营地不再使用固定的水、食物和木材阈值。每次规划都会计算未来三日安全库存：

```text
水目标 = 人口 × 三日饮水 × 温度倍率
食物目标 = 人口 × 三日进食 × 季节倍率 × 腐败缓冲
木材目标 = 三日燃料 + 未预留建造需求 + 人口缓冲
```

计算还会处理：

- 当前季节、天气和气温。
- 食物腐败风险与储物棚保护。
- 当前储存容量；初始 24 容量会把自然目标压缩到可实现预算，储物棚建成后恢复完整三日目标。
- 营地现货、人物背包和已分配采集任务的预计产出。
- 工地材料预留和正在执行的添柴承诺。
- 同一规划轮内刚分配的在途资源，避免后续人物重复接单。

当动态目标已经满足时，非紧急村民不再继续创建取水、采集或伐木任务。个人严重饥渴仍可越过聚落库存目标执行紧急生存行动。

营地面板显示：

```text

```

## 目录

```text
version.json      # GitHub Pages 部署清单与构建标记
.github/workflows # CI：语法、测试、确定性回放与生产构建
src/
  core/
    simulation/   # 固定步长时钟
    ui/           # UI 合并与限频调度
    ids/          # 可复现 ID 序列
  modules/
    people/       # 人物、轻量运行时视图、关系、库存与记忆
    map/          # 地图生成、查询与服务层
    actions/      # 规划、动态库存、固定 tick 与统一预留账本
    settlements/  # 营地库存、容量、规则与储存升级
    buildings/    # 工地、施工、居住与储物棚
    environment/  # 昼夜、天气、篝火与暴露
    ecology/      # 树木与浆果恢复
    roads/        # tick 驱动的踩踏与土路
    seasons/      # 四季日历与农业窗口
    farming/      # 粟米、农田、扩田与土壤
    storage/      # 食物批次、新鲜度与损耗
    persistence/  # 世界快照、运行时坐标与事务化读档
  bootstrap/      # 浏览器运行时扩展和各模块 UI
  ui/map/         # 地图渲染与交互
  styles/         # 桌面与移动端样式
```

## 已知限制

- 存档保存在当前浏览器，没有跨设备云存档。
- 读档恢复人物精确位置，但不会续接原任务的路径游标、工作耗时或中途动画；未完成任务会取消后重新规划。
- 在途采集量当前按行动类型的标准产量估算，第二阶段劳动成本接入后再改为任务级精确预测。
- 动态库存参数仍属于原型平衡值，需要第 60 日长期回归校准。
- 当前账本尚未形成完整资源收支流水和每日结算报表。
- 草棚、储物棚和农田尚未成为寻路障碍。
- 真实 iPhone 的安全区、浏览器地址栏变化和极小屏可读性仍需人工回归。

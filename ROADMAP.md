# 《生灵》机制优化与开发规划

## 总目标

《生灵》要成为一个可长期运行、可解释、会自然产生历史的规则驱动世界。任何人物传记、关系变化、史书和未来 AI 对话都必须来自已经发生的世界事实。

## 机制调整原则

1. **先约束，再评分。** 行动规划先检查路线、库存、季节、资源和任务名额等硬条件，再进行效用评分。
2. **所有占用都要预留。** 营地容量、工地建材、资源目标、工具和任务名额使用可释放的预留记录。
3. **个人与聚落分账。** 人物背包、家庭储备、营地公共库存、工地库存分开，所有转移进入结构化流水。
4. **世界时间独立于画面刷新。** 模拟使用固定步长推进，UI 限频渲染；10× 增加模拟吞吐，不增加无意义 DOM 更新。
5. **危机必须有物质原因。** 饥荒来自产量、季节、运输、分配和腐败；冲突来自资源与关系。
6. **AI 只负责表达。** AI 可以润色史书、续写传记、扮演人物回答已知事实，不能决定行动或修改世界状态。

## 已完成底座

### 固定模拟步长 · v0.26 ✅

- 每个固定 tick 代表 1 个世界分钟。
- `0.5×—10×` 只决定每个现实帧消费多少 tick。
- 季节、天气、篝火、生态、道路、农业、食物、行动和需求按固定事件顺序结算。
- UI 与模拟解耦，主要界面最高 10 FPS。
- 同一种子第 30 日回放使用固定 SHA-256 指纹校验。

### 行动与资源预留底座 · v0.26 ✅

- 任务名额、资源物件、营地容量、工地材料和工具进入统一运行时预留账本。
- 任务取消、人物死亡、路线失败、完成和读档重建会释放预留。
- 固定种子 ID 序列覆盖人物、任务、建筑、记忆和日志。
- 高频行动使用轻量人物运行时视图，避免复制完整记忆历史。

### 动态目标库存 · v0.27.1 ✅

- 目标库存根据人口和未来三日基础消耗计算。
- 水目标读取气温；食物目标读取季节、腐败风险和储存保护；木材目标读取季节、天气、篝火与未预留建造需求。
- 初始小容量营地按容量预算压缩目标，储物棚建成后恢复完整目标。
- 有效库存包含营地现货、人物背包和已分配采集任务的预计产出，并扣除工地材料与添柴承诺。
- 同一规划轮的新任务立即进入在途估算，避免多人继续追逐已经被覆盖的缺口。

### 劳动成本模型 · v0.27.2 ✅

- 采集、搬运、施工、农事和添柴共用统一劳动成本快照。
- 距离、实际 A* 路线、负重、地形、道路、天气、精力、技能和行动强度共同决定预计耗时与能耗。
- 候选评分使用预估成本，任务分配后使用实际路线生成精确 `task.data.laborCost`。
- 额外能耗在任务阶段内累计、完成时一次结算；移动热路径不复制完整人物或地图对象。

### 工具与耐久 · v0.27.3 ✅

- 营地初始拥有石斧、搬运篮、简易农具和石镐四件公共工具。
- 石斧作用于伐木，搬运篮作用于搬运与建材运输，农具作用于开垦、播种和收获。
- 工具效果直接进入劳动成本快照。
- 同一工具不能同时被两个任务占用。
- 任务完成扣耐久；取消、死亡、路线失败或孤立任务只释放占用。
- 工具耐久进入长期存档，瞬时占用不持久化；失败读档恢复工具检查点。

### 统一资源流水 · v0.27.4 ✅

- 人物背包、营地库存与工具耐久进入统一账户流水。
- 每笔记录包含 tick、世界时间、物品、数量、单位、来源、去向、类别、原因、人物、任务、预留和元数据。
- 同一 tick 内，属于同一任务或明确搬运链的账户增减配对为一笔内部转移。
- 独立生产和消费不会因发生在同一 tick 而错误抵消。
- 未配对增加归入生产源；未配对减少归入需求、篝火、工地、腐败或其他消费去向。
- 流水保留最近 5,000 笔，进入长期存档与失败读档检查点。
- `verify()` 自动检查重复记录、非法流向、负库存和工具耐久越界。

### 每日经济摘要与第 60 日回归 · v0.27.5 ✅

- 每个世界日保存期初、期末、生产、消费、燃料、施工、腐败、内部转移和净变化。
- 按物品校验“实际库存变化”与“流水预期变化”的差异。
- 记录劳动任务分配、完成、预计耗时、预计精力和行动分类。
- 记录食物与饮水请求被拒次数及原因。
- 结合三日目标库存识别供给缺口。
- 自动识别生存短缺、腐败压力、劳动积压、库存缺口和账实差异。
- 日报进入长期存档；旧存档从当前世界状态建立当日草稿；失败读档恢复日报检查点。
- 保留第 30 日世界基线，并新增第 60 日生存经济基线。
- 1×、5×、10× 批次推进在相同世界时间必须得到相同世界指纹。

第 60 日固定基线：

```text
seed: replay-seed-v0275-day60
target: 生灵历 1 年第 60 日 12:00
fixed ticks: 85,200
SHA-256: 68cc6feff5e715fd21d6386e199d7876a11d01d5f87cff31a58014d33cd1584b
```

## v0.28 农业与家庭

### 农业闭环

- 加入可计量种子储备、播种消耗和播种失败。
- 加入休耕、堆肥、水分和多作物。
- 每块田记录水分、肥力、作物、劳动投入和历史产量。
- 扩田需要劳动力预算与地形评分。
- 收获进入人物背包，再运输到公共或家庭储备。

### 正式维修经济

- 把工具修理和替换从直接 API 升级为任务。
- 修理需要材料、工具、劳动时间和工作地点。
- 维修失败、材料不足和工具等待进入资源流水与日报。

### 家庭账户

- 从公共营地逐步演化出家庭、工队和公共职务。
- 家庭拥有独立食物、工具和居住资源。
- 公共库存与家庭库存通过结构化转移流水连接。
- 分配规则开始影响真实库存，而不只生成解释文本。

## 后续机制优化

### 社会结构

- 规则由资源危机和既有关系触发提议、支持、反对与执行。
- 关系变化对应具体事件，并设置每日变化上限。
- 交易、拒绝、偏袒、共同劳动和救助形成可追溯社会事实。

- 动态目标库存。
- 劳动成本模型。
- 工具与耐久。

- 种子、堆肥、休耕、水分和多作物。

- 家庭关系转为家庭库存与共同决策。

### v0.29 · 规则、冲突与治理

- 规则提议、表态、执行和修订。
- 职务、工队、资源分配与惩罚。
- 冲突、调解、分裂和迁徙全部由事实链驱动。

### v0.30 · 传记、史书与人物对话

- 固定传记首次生成，后续只追加。
- 史书按可选周期编号并锁定。
- 人物对话只读取本人亲历、听闻、关系和公共事实。
- DeepSeek 接口保持手动触发，不进入模拟循环。

## 每版完成标准

- 语法检查和单元测试全部通过。
- 10× 至少运行到目标世界日，无模拟暂停。
- 没有孤立预留、负库存、重复交付或无法解释的资源增长。
- 新存档可读取，上一版存档有明确迁移路径。
- 桌面和手机的主操作完成实测。
- `codex.md`、`version.json` 和 README 同步更新。

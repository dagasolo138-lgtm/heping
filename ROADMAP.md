# 《生灵》机制优化与开发规划

## 总目标

《生灵》要成为一个可长期运行、可解释、会自然产生历史的规则驱动世界。任何人物传记、关系变化、史书和未来 AI 对话都必须来自已经发生的世界事实。

## 机制调整原则

1. **先约束，再评分。** 行动规划先检查路线、库存、季节、资源和任务名额等硬条件，再进行效用评分，避免高分任务实际上无法执行。
2. **所有占用都要预留。** 营地容量、工地建材、农田劳动力、建筑位置和未来交易物资统一使用可释放的预留记录。
3. **个人与聚落分账。** 人物背包、家庭储备、营地公共库存、工地库存必须分开，所有转移写入结构化流水。
4. **世界时间独立于画面刷新。** 模拟使用固定步长推进，UI 限频渲染；10× 只增加模拟吞吐，不增加无意义 DOM 更新。
5. **危机必须有物质原因。** 饥荒来自产量、季节、运输、分配和腐败；冲突来自资源与关系，不使用随机文本直接制造剧情。
6. **AI 只负责表达。** AI 可以润色史书、续写传记、扮演人物回答已知事实，不能决定行动或修改世界状态。

## 近期机制优化

### 1. 行动与资源账本

- 为采集、搬运、施工、进食建立统一的 reservationId 和资源流水。
- 任务取消、人物死亡、存档读取、路线失败时自动释放预留。
- 日志显示“计划量 / 实际量 / 失败原因”，减少无法解释的状态。
- 每个世界日生成资源收支摘要：采集、消耗、施工、腐败、结余。

### 2. 固定模拟步长

- 将世界更新改为固定 tick，例如每 tick 代表 1 个世界分钟。
- 0.5×—10× 只决定每个现实帧处理多少 tick。
- UI 最高 8—10 次/秒刷新，日志批量追加。
- 加入同一种子重放测试，保证相同输入得到相同结果。

### 3. 生存与劳动闭环

- 人物根据体力、饥渴、天气、距离和技能估算劳动成本。
- 长距离搬运消耗更多精力，土路和工具降低成本。
- 低健康、严重受寒、极端疲劳会停止高强度劳动。
- 营地目标库存根据人口、季节和未来三日消耗动态计算。

### 4. 农业闭环

- 先加入休耕、堆肥、种子储备和播种失败，再增加新作物。
- 每块田记录水分、肥力、作物、劳动投入和历史产量。
- 扩田需要劳动力预算与地形评分，不能只靠固定坐标触发。
- 收获进入人物背包，再运输到公共或家庭储备。

### 5. 社会结构

- 从公共营地逐步演化出家庭、工队和公共职务。
- 规则由资源危机和既有关系触发提议、支持、反对与执行。
- 关系变化必须对应具体事件，并设置每日变化上限。
- 交易、拒绝、偏袒、共同劳动和救助形成可追溯社会事实。

## 版本路线

### v0.25 · 地图主导观察器

- 桌面端地图主界面与右侧标签观察栏。
- 手机端地图首屏与底部抽屉。
- 人物、营地、事件、史书分层。
- 存档和构建信息收进系统菜单。

### v0.26 · 确定性模拟内核

- 固定 tick、UI 限频、日志聚合。
- 统一资源与任务预留账本。
- 同种子重放与第 30 日稳定性测试。

### v0.27 · 生存经济

- 动态目标库存、劳动成本、工具耐久。
- 资源收支日报与瓶颈提示。
- 食物、水、燃料、住房形成完整约束链。

### v0.28 · 农业与家庭

- 堆肥、休耕、水分、种子和多作物。
- 家庭关系转为家庭库存与共同决策。
- 公共库存与私人库存开始分化。

### v0.29 · 规则、冲突与治理

- 规则提议、表态、执行和修订。
- 职务、工队、资源分配与惩罚。
- 冲突、调解、分裂和迁徙全部由事实链驱动。

### v0.30 · 传记、史书与人物对话

- 固定传记首次生成，后续只追加。
- 史书按可选周期编号并锁定。
- 人物对话只读取本人亲历、听闻、关系和公共事实。
- DeepSeek 接口保持手动触发，不进入模拟循环。

## 每版完成标准

- 语法检查和单元测试全部通过。
- 10× 至少运行到目标世界日，无模拟暂停。
- 没有孤立预留、负库存、重复交付或无法解释的资源增长。
- 新存档可读取，上一版存档有明确迁移路径。
- 桌面和手机的主操作均完成实测。
- codex.md、version.json 和 README 同步更新。

# 《生灵》机制优化与开发规划

## 总目标

《生灵》要成为一个可长期运行、可解释、会自然产生历史的规则驱动世界。任何人物传记、关系变化、史书和未来 AI 对话都必须来自已经发生的世界事实。

## 机制调整原则

1. **先约束，再评分。** 行动规划先检查路线、库存、季节、资源和任务名额等硬条件，再进行效用评分。
2. **所有占用都要预留。** 营地容量、工地建材、资源目标和任务名额使用可释放的预留记录。
3. **个人与聚落分账。** 人物背包、家庭储备、营地公共库存、工地库存分开，所有转移最终进入结构化流水。
4. **世界时间独立于画面刷新。** 模拟使用固定步长推进，UI 限频渲染；10× 增加模拟吞吐，不增加无意义 DOM 更新。
5. **危机必须有物质原因。** 饥荒来自产量、季节、运输、分配和腐败；冲突来自资源与关系。
6. **AI 只负责表达。** AI 可以润色史书、续写传记、扮演人物回答已知事实，不能决定行动或修改世界状态。

## 已完成底座

### 固定模拟步长 · v0.26

- 每个固定 tick 代表 1 个世界分钟。
- `0.5×—10×` 只决定每个现实帧消费多少 tick。
- 季节、天气、篝火、生态、道路、农业、食物、行动和需求按固定事件顺序结算。
- UI 与模拟解耦，主要界面最高 10 FPS。
- 同一种子第 30 日回放使用固定 SHA-256 指纹校验。

### 行动与资源预留底座 · v0.26

- 任务名额、资源物件、营地容量和工地材料映射进入统一运行时预留账本。
- 任务取消、人物死亡、路线失败、完成和读档重建会释放预留。
- 固定种子 ID 序列覆盖人物、任务、建筑、记忆和日志。
- 高频行动使用轻量人物运行时视图，避免复制完整记忆历史。

## 近期机制优化

### 1. 生存与劳动闭环

- 人物根据体力、饥渴、天气、距离和技能估算劳动成本。
- 长距离搬运消耗更多精力，土路和工具降低成本。
- 低健康、严重受寒、极端疲劳停止高强度劳动。
- 营地目标库存根据人口、季节和未来三日消耗动态计算。

### 2. 资源流水与日报

- 为采集、搬运、施工、进食和腐败建立统一资源流水。
- 日志记录计划量、实际量和失败原因。
- 每个世界日生成资源收支摘要：采集、消耗、施工、腐败和结余。
- 从账本计算瓶颈、积压、空跑与供给缺口。

### 3. 农业闭环

- 加入休耕、堆肥、种子储备和播种失败，再增加新作物。
- 每块田记录水分、肥力、作物、劳动投入和历史产量。
- 扩田需要劳动力预算与地形评分。
- 收获进入人物背包，再运输到公共或家庭储备。

### 4. 社会结构

- 从公共营地逐步演化出家庭、工队和公共职务。
- 规则由资源危机和既有关系触发提议、支持、反对与执行。
- 关系变化对应具体事件，并设置每日变化上限。
- 交易、拒绝、偏袒、共同劳动和救助形成可追溯社会事实。

## 版本路线

### v0.25 · 地图主导观察器 ✅

- 桌面端地图主界面与右侧标签观察栏。
- 手机端地图首屏与底部抽屉。
- 人物、营地、事件、史书分层。
- 存档和构建信息收进系统菜单。

### v0.26 · 确定性模拟内核 ✅

- 固定 tick 与固定结算顺序。
- UI 10 FPS 合并调度。
- 统一运行时预留账本。
- 可复现 ID 序列。
- 第 30 日确定性世界指纹。
- GitHub Actions 自动检查、测试和构建。

### v0.27 · 生存经济

- 动态目标库存、劳动成本、工具耐久。
- 资源收支日报与瓶颈提示。
- 食物、水、燃料、住房形成完整约束链。

### v0.28 · 农业与家庭

- 堆肥、休耕、水分、种子和多作物。
- 家庭关系转为家庭库存与共同决策。
- 公共库存与私人库存开始分化。

### v0.29 · 规则、冲突与治理

- 规则提议、表态、执行和修订。
- 职务、工队、资源分配与惩罚。
- 冲突、调解、分裂和迁徙全部由事实链驱动。

### v0.30 · 传记、史书与人物对话

- 固定传记首次生成，后续只追加。
- 史书按可选周期编号并锁定。
- 人物对话只读取本人亲历、听闻、关系和公共事实。
- DeepSeek 接口保持手动触发，不进入模拟循环。

## 每版完成标准

- 语法检查和单元测试全部通过。
- 10× 至少运行到目标世界日，无模拟暂停。
- 没有孤立预留、负库存、重复交付或无法解释的资源增长。
- 新存档可读取，上一版存档有明确迁移路径。
- 桌面和手机的主操作完成实测。
- `codex.md`、`version.json` 和 README 同步更新。
